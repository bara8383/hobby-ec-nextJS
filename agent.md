# Agent運用ガイド（プロジェクト方針）

このドキュメントは、このプロジェクトでタスクを実装するエージェント向けの運用ガイドです。

## プロジェクト優先順位

1. **Next.jsの最新設計思想を取り込む（公式情報を最優先で参照）**
   - 参照先（必須）:
     - https://nextjs.org/docs
     - https://nextjs.org/blog
   - 実装方針:
     - App Router / Server Components / Server Actions など、最新の推奨パターンを優先。
     - 非推奨APIや旧来の実装パターンは新規採用しない。

2. **SEOを高度に最適化する**
   - メタデータ、構造化データ、OGP、サイトマップ、robots、canonical、内部リンク設計を重視。
   - Core Web Vitals の改善に寄与する設計を優先。

3. **リアルタイムチャット機能を実装する**
   - UX・可用性・保守性を重視し、段階的に導入する。
   - 必要に応じて接続方式（WebSocket/SSE等）と運用コストを比較して採択する。

4. **AWSでデプロイ（コスト最小化）**
   - 無駄な常時稼働リソースを避ける。
   - 初期は小さく始め、アクセス増加に応じてスケールする。
   - 構成選定時は「月額見積り」と「運用負荷」を合わせて評価する。

## タスク実装時の必須対応

### 1. PR本文に以下をわかりやすく詳細に記載する（なるべく日本語）

1.1 **何を実装・修正したか**
- 変更点を箇条書きで具体的に記載する。
- 影響範囲（画面、API、インフラ、ドキュメント等）を明記する。

1.2 **なぜその実装にしたか（理由）**
- 代替案があれば比較し、採用理由を説明する。
- 優先順位（最新Next.js / SEO / チャット / AWSコスト）との整合性を明記する。

### 2. 実装中に感じた提案・懸念点・問題点を `/docs/proposals.md` に記載する

- 記載目的: オーナーが方針判断できるようにするため。
- 記載粒度:
  - 提案の背景
  - メリット / デメリット
  - 推奨アクション
  - 優先度（High / Medium / Low）
