# AGENTS.md

> この文書は、Codex へ実装依頼する際に最初に読み込まれる前提の運用ガイドです。

## 0. クイックチェック（実装開始前）

- [ ] 優先順位（1〜4）を確認した
- [ ] 変更前に作業ブランチを作成した
- [ ] PR に「実装内容」「実装理由」「提案・懸念点」を記載する準備をした
- [ ] 実装中に気づいた提案・懸念点・問題点を `/docs/proposals.md` へ記録する

## 1. プロジェクトの優先順位

1. Next.js の最新の設計思想・実装パターンを採用する（必ず公式ドキュメントを参照する）。
2. SEO を高度に最適化する。
3. リアルタイムチャット機能を実装する。
4. AWS にデプロイする。ただしコストは最小化する。

## 2. タスク実装時の必須ルール

プロジェクトオーナーから実装タスクを受けた際は、必ず以下を実施すること。

1. PR に、次の内容をわかりやすく詳細に記載する（なるべく日本語）。
   - 何を実装・修正したか
   - なぜその実装方針にしたか（判断理由）
2. 実装中に気づいた提案・懸念点・問題点は、`/docs/proposals.md` に記載する。
   - 最終判断はプロジェクトオーナーが行う。
3. PR を作成するため、リポジトリに変更を入れる前に、必ず作業ブランチを作成する。

## 3. PR 記載テンプレート（推奨）

- 実装・修正内容:
  - ...
- 実装理由:
  - ...
- 提案・懸念点:
  - ...
