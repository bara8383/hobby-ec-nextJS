# P-030: Mitigate Per User Sqs Queue Scaling Risk

- Category: concern
- Priority: high
- Status: pending
- Related: 

---

## 概要
`lib/chat/queue.ts` で採用した「ユーザー単位キュー方式」は、アクティブユーザー増加時に SQS キュー数クォータや CreateQueue API スロットリングの影響を受ける可能性を確認。【対応案】運用フェーズでキュー上限監視を追加し、しきい値超過時は「ユーザーシャード単位キュー + userId属性振り分け」へ段階移行する。

## 背景（Why）
- 起票カテゴリ: Concern / Architecture
- 対象スコープ: BE
- 期待影響: ユーザー増加時に新規キュー作成遅延が発生し、初回受信レイテンシ悪化やイベント取りこぼしリスクが増加。

- 現状: `lib/chat/queue.ts` で採用した「ユーザー単位キュー方式」は、アクティブユーザー増加時に SQS キュー数クォータや CreateQueue API スロットリングの影響を受ける可能性を確認。【対応案】運用フェーズでキュー上限監視を追加し、しきい値超過時は「ユーザーシャード単位キュー + userId属性振り分け」へ段階移行する。
