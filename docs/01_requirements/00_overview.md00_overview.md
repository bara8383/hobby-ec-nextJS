# 00_overview

## 1. 目的
本プロジェクトは、デジタル商品を中心としたCtoCマーケットプレイス型ECを構築し、実務レベルの要件定義〜設計・実装・運用までを学習することを目的とする。

※本ドキュメントは「要件定義」であり、実装方式・技術選定は扱わない。

## 2. 対象範囲（Phase）
### Phase1（MVP）
- CtoCマーケットプレイス（複数出品者）
- デジタル商品（ダウンロード提供）
- 会員登録必須（ゲスト購入なし）
- 運営による返金一元対応
- 週次精算、手数料あり
- 初回出品のみ審査
- 通報・停止の最低限対応

### Phase2（将来）
- 物販の追加（配送・在庫等）
- その他の高度機能（検索強化、レコメンド等）

## 3. 前提・方針
- UXは基本「高い方」を優先する（ただし運用破綻を避ける）
- 要件と設計を混在させない
- 未決事項は risks に隔離する

## 4. 用語
- 購入者：商品を購入しダウンロードする会員
- 出品者：商品を出品（登録・編集・販売停止）する会員
- 運営者：審査、返金、通報対応、停止判断、精算管理を行う管理者
- 注文：購入者の購入単位（複数商品をまとめ買い可能）
- DLライブラリ：購入者が購入済み商品を確認しDLする場所（取引情報は原則表示しない）
- 注文履歴：購入者が取引情報（購入日・金額・状態等）を確認する場所
