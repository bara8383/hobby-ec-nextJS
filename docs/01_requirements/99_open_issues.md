# 99_open_issues (risks)

本章は未決事項・前提揺れ・リスクを要件から分離して記録する。
各項目は「優先度」「意思決定者」「判断期限（目安）」「暫定方針」を持ち、要件に混入させない。

凡例：
- 優先度：P0（今決めないと前に進まない）/ P1（基本設計までに必要）/ P2（詳細設計まででよい）/ P3（運用しながら改善）
- 意思決定者：購入者UX責任/運営責任/出品者UX責任/プロダクト責任（=あなた）など
- 判断期限（目安）：要件凍結・基本設計着手・詳細設計着手などのマイルストン

---

## R-01 禁止商材の具体リスト
- 優先度：P1
- 意思決定者：運営責任 + プロダクト責任（あなた）
- 判断期限（目安）：基本設計着手まで
- 背景/リスク：
  - 初回出品審査・通報対応の判断基準が曖昧だと、審査品質がブレて炎上・不正流入の温床になる
- 決定が必要な点：
  - 禁止カテゴリの定義（例：権利侵害を助長する素材、危険行為を促す内容、明らかな詐欺等）
  - 禁止判断の公開範囲（利用規約にどこまで明記するか）
- 暫定方針：
  - 「権利侵害が明確/強く疑われる」「危険行為の助長」「詐欺・虚偽説明」を禁止の最小軸として開始

## R-02 返金例外（不正/悪用/規約違反）の判定基準
- 優先度：P1
- 意思決定者：運営責任 + プロダクト責任（あなた）
- 判断期限（目安）：基本設計着手まで
- 背景/リスク：
  - 返金は原則「未DLのみ可」だが、例外を個別対応にしたため、判断軸がないと運営コストが爆増する
- 決定が必要な点：
  - 例外扱いの代表ケース（不正注文、悪用、規約違反、通報絡み等）
  - 「拒否/保留/個別対応」の区分ルール
  - 必要最低限の証跡（ログ等）を運用でどう扱うか
- 暫定方針：
  - 返金は自動確定せず、疑いがあれば「個別対応（原則拒否）」に分岐できる前提で運用開始

## R-03 通報対応のSLA（対応期限の約束）
- 優先度：P2
- 意思決定者：運営責任 + プロダクト責任（あなた）
- 判断期限（目安）：詳細設計着手まで
- 背景/リスク：
  - SLAを要件に含めると運営体制の縛りになる。含めないと利用者の不信を招く可能性がある
- 決定が必要な点：
  - 「受付完了」「初動」「最終判断」の目安を対外的に持つか
  - 夜間休日の扱い
- 暫定方針：
  - MVPではSLAは外部約束しない（内部運用目安は後で決める）

## R-04 出品者停止の条件・期間・解除
- 優先度：P1
- 意思決定者：運営責任 + プロダクト責任（あなた）
- 判断期限（目安）：基本設計着手まで
- 背景/リスク：
  - 「重大/再犯」の定義や停止期間が曖昧だと、恣意性・不公平感・トラブルが発生する
- 決定が必要な点：
  - 即時停止/段階停止のライン
  - 停止期間（一定期間/無期限/段階）
  - 解除条件（再発防止・説明・修正等）
- 暫定方針：
  - 重大・再犯は停止可能、解除条件は運営が個別判断（MVPは運用で回し、後で明文化）

## R-05 通知テンプレ（定型文）と簡易理由の粒度
- 優先度：P2
- 意思決定者：運営責任
- 判断期限（目安）：詳細設計着手まで
- 背景/リスク：
  - 自由記述を許すと運営品質がブレる。定型のみだと説明不足で問い合わせが増える
- 決定が必要な点：
  - 定型文中心にするか、自由記述を許すか
  - 「簡易理由」のテンプレ粒度（通報者/出品者向け）
- 暫定方針：
  - 定型文中心＋必要に応じて短い追記（運用で調整）

## R-06 支払い手段の具体一覧（決済区分の確定）
- 優先度：P1
- 意思決定者：プロダクト責任（あなた）
- 判断期限（目安）：基本設計着手まで
- 背景/リスク：
  - 期限（即時決済1時間/後払い24時間）は区分で決めたが、具体手段が決まらないと購入フロー要件が固まりきらない
- 決定が必要な点：
  - 採用する支払い手段の一覧
  - それぞれの区分（即時決済/後払い等）
- 暫定方針：
  - 区分の要件は固定し、具体手段は設計開始前に確定する

## R-07 未読通知の蓄積（未読は自動削除しない）
- 優先度：P3
- 意思決定者：購入者UX責任 + 運営責任
- 判断期限（目安）：運用しながら
- 背景/リスク：
  - 未読が増え続けるとUX劣化や一覧性能に影響する可能性がある
- 決定が必要な点：
  - 上限件数、古い未読の扱い、未読フィルタ等の要否
- 暫定方針：
  - MVPは現状ルール（未読は自動削除しない＋ユーザー削除可）で開始し、運用データで判断

## R-08 退会後データの保持方針（内部保持/匿名化/削除）
- 優先度：P2
- 意思決定者：運営責任 + プロダクト責任（あなた）
- 判断期限（目安）：詳細設計着手まで
- 背景/リスク：
  - ユーザーは退会後に閲覧不可だが、取引証跡や運営対応のため内部保持が必要になる可能性がある
- 決定が必要な点：
  - 内部保持期間、匿名化範囲、削除範囲
  - 返金・通報など運営対応時の照会可否
- 暫定方針：
  - ユーザー閲覧は不可のまま、内部保持は運用上必要な範囲で保持（具体は後で決める）
