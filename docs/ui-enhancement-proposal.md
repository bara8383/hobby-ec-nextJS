# 画面作り込み提案（初版）

## 目的

「Digital Creator Market」の主要画面を、MVPの機能を維持したまま**情報設計・視認性・購入導線**の観点で強化し、以下を同時に満たす。

1. Next.js 最新設計（App Router / Server Components中心）との整合
2. SEO強化（構造化データ・内部リンク・説明テキスト最適化）
3. リアルタイムチャット導線の自然な統合
4. AWS最小コスト運用を前提にした実装コスト抑制

---

## 現状課題（repo確認ベース）

- トップページが「ヒーロー + 商品グリッド + タグ」の最小構成で、購入意思決定に必要な比較情報が薄い。
- 商品一覧/商品詳細への導線はあるが、ランキング・特集・利用シーン別導線がない。
- チャットはフローティングで使えるが、「どの段階で相談すべきか」の文脈が画面上に不足。
- UIトーンがほぼ単一で、差別化要素（カテゴリ・価格帯・即時DL可否）が一目で伝わりにくい。

---

## 提案方針

### A. ホーム画面の情報階層を3層化

1. **ファーストビュー**
   - キャッチコピー + CTA（商品一覧へ / カテゴリを見る）
   - 「即時ダウンロード」「商用利用可」「チャット相談可」の3つの価値訴求バッジ
2. **探索支援ブロック**
   - カテゴリカード（画像 or アイコン）
   - 人気タグクラウド
   - 価格帯ショートカット（〜¥1,000 / ¥1,001〜¥3,000 / ¥3,001〜）
3. **商品訴求ブロック**
   - 新着商品
   - 人気商品（疑似ランキングでも可）
   - 用途別特集（例: YouTubeサムネ向け、配信BGM向け）

### B. 商品カードの情報密度を最適化

- 表示項目を「サムネイル / タイトル / 価格 / カテゴリ / タグ（先頭2件）/ サンプル有無」に整理
- CTAを「詳細を見る」に統一し、カード全体クリックは維持
- 在庫概念がないデジタル商品特性を活かし「即時DL」ラベルを標準表示

### C. 商品詳細の購買支援UIを強化

- 右カラム固定（価格 / Add to Cart / 購入前FAQリンク / チャット起動）
- 下部に「関連商品」「同タグ商品」を追加し回遊導線を増強
- SEO向けにFAQ構造化データ（Q&A）を段階導入

### D. チャット導線の文脈付け

- ヘッダー近傍・商品詳細・カートに「購入前の相談」導線を配置
- 初回メッセージにテンプレート候補を表示（例: 利用範囲確認 / 納品形式確認）
- 画面遷移後も会話を維持（現行Widget方針を踏襲）

### E. デザインシステム最小導入

- `app/globals.css` にトークン（spacing/color/radius/shadow）を定義
- コンポーネントは `ProductCard` / `ChatWidget` / `Button` 相当を共通化
- 実装はCSS Modulesに寄せず、現行方針を踏襲して最小差分で改善

## 画面別ワイヤー（テキスト）

### 1) Home

- Hero
- Value badges (3)
- Category cards (4)
- New arrivals
- Popular picks
- Tags
- Chat CTA band

### 2) Products一覧

- パンくず
- 見出し + 説明文（SEO文）
- Filter（カテゴリ/価格/タグ）
- ソート（新着/価格/人気）
- 商品グリッド

### 3) Product詳細

- メイン画像 + 概要
- スペック（解像度/BPM/尺などカテゴリ別）
- 固定購入パネル
- FAQ
- 関連商品

### 4) Cart / Checkout

- ステップ表示（Cart → Checkout → Complete）
- 安心訴求（決済安全性 / ダウンロード手順）
- チャットへの補助導線

## 実装ステップ案

### Phase 1（低コスト・短納期）

- Homeのセクション追加（ヒーロー強化、カテゴリカード、特集枠）
- ProductCardの情報整理
- CTA文言統一

### Phase 2（CV改善）

- Product詳細の購入支援UI追加
- 関連商品導線追加
- FAQ（画面表示 + JSON-LD）

### Phase 3（運用改善）

- 簡易ランキングロジック（閲覧数/購入数を将来連携）
- A/Bテストしやすいセクション分割
- AWS移行時にキャッシュ方針（ISR/Edge）最適化

---

## 受け入れ基準（提案）

- 主要導線（Home→一覧→詳細→カート）で視覚的に次アクションが常に1つ以上示される
- HomeのSEO説明文と内部リンクが増え、カテゴリ/タグLPへの到達が明確になる
- 商品詳細で「購入前相談」と「Add to Cart」がスクロール中も迷わず操作できる
- UI拡張後も既存機能（カート/チェックアウト/チャット）が壊れない

---

## 懸念と先回り対応

- 懸念: 一度に作り込みすぎると差分が大きくレビューしづらい
  - 対応: Phase分割でPRを小さく保つ（1PR 1テーマ）
- 懸念: 画像素材が未整備だと見た目改善が限定的
  - 対応: 先行してプレースホルダーとタイポグラフィ改善を優先
- 懸念: チャットの利用増加で応答品質が課題化
  - 対応: FAQテンプレート表示で定型質問を吸収し、運用負荷を抑える
