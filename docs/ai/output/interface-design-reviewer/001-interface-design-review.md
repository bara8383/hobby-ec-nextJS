# 2. インターフェースデザイン総評

**判定: 要調整**

EC サイトとして必要な主要画面と基本 UI 部品は揃っており、商品探索、商品詳細確認、カート、チェックアウト、チャット相談までの画面内構造は概ね成立している。特に、商品一覧では左側フィルタと右側商品グリッドの 2 カラム構造、モバイル時の 1 カラム化、商品カードの CTA 固定により、基本的な探索インターフェースは理解しやすい。商品一覧の 2 カラム構造とモバイル時の 1 カラム化は CSS 上でも定義されている。 【F:app/globals.css†L377-L386】【F:app/globals.css†L1381-L1389】

一方で、購入判断に必要な情報の優先順位、購入フロー内の確認密度、チャット画面の状態表示、商品カードの情報量には改善余地がある。商品カードには商品画像、商品名、価格、詳細 CTA はあるが、カテゴリ、ライセンス、ファイル形式、商用利用可否、納品形式など、EC の比較判断で必要になりやすい情報がカード上に少ない。 【F:components/ProductCard.tsx†L31-L58】

カートとチェックアウトは最短導線としては機能するが、注文確定前の画面内情報がかなり簡素で、商品単価、数量、合計、購入後の入手方法、ダウンロード期限、ライセンス確認などの対応関係が弱い。カートでは価格と数量は表示されるが、チェックアウトでは商品名と数量だけのリストになっている。 【F:components/cart/CartContent.tsx†L37-L54】【F:components/checkout/CheckoutContent.tsx†L70-L83】

チャットは接続状態、エラー、既読、入力欄、送信 disabled を持ち、リアルタイム相談の画面部品は揃っている。ただし、メッセージが 0 件の empty state、再認証時の次アクション、再接続中の待機表示、送信失敗後の回復操作が画面上で十分に構造化されているとは言いにくい。 【F:components/chat/LineLikeChat.tsx†L251-L280】

# 3. 良好なインターフェース設計

## 3.1 商品一覧の探索構造が明確

商品一覧は、フィルタ領域と商品一覧領域が分離されており、デスクトップでは左に条件、右に結果という EC で一般的な探索構造になっている。CSS では `grid-template-columns: minmax(250px, 280px) minmax(0, 1fr)` が指定され、フィルタと結果領域の役割が明確に分かれている。 【F:app/globals.css†L377-L386】

## 3.2 商品件数とソートが一覧上部に配置されている

商品一覧では、商品件数とソートが商品グリッドの上部にまとまっている。ユーザーは現在の条件で何件表示されているかを確認しながら並び替えられる。 【F:app/(store)/products/page.tsx†L72-L86】

## 3.3 empty state が主要画面に存在する

商品一覧、カート、チェックアウトには empty state がある。商品一覧では条件不一致時の説明があり、カートとチェックアウトでは次に戻るべき導線が提示される。 【F:app/(store)/products/page.tsx†L78-L80】【F:components/cart/CartContent.tsx†L22-L29】【F:components/checkout/CheckoutContent.tsx†L57-L64】

## 3.4 商品カードの基本構造が一貫している

商品カードは、画像、商品名、価格、詳細 CTA の順で並び、カード単位で比較しやすい。CTA はカード末尾にあり、カードごとの行動が明確である。 【F:components/ProductCard.tsx†L31-L58】

## 3.5 カートで商品単位の操作と合計が分離されている

カートでは、商品行ごとの情報・数量操作と、画面下部の合計・チェックアウト導線が分離されている。商品単位の調整と購入へ進む操作を別のまとまりとして理解できる。 【F:components/cart/CartContent.tsx†L34-L55】

## 3.6 チャット画面に接続状態とエラー表示がある

チャット詳細では、ヘッダーに接続状態が表示され、エラーがある場合は alert として表示される。リアルタイム機能で重要な「今送れる状態か」を画面内である程度判断できる。 【F:components/chat/LineLikeChat.tsx†L251-L260】

## 3.7 モバイル向けの主要再配置が用意されている

商品一覧はモバイル時に 1 カラム化され、ツールバーは縦並びになる。チャットフォームもモバイル時に縦並びになり、入力欄と送信ボタンが狭い横幅で詰まりにくい。 【F:app/globals.css†L1381-L1408】【F:app/globals.css†L1423-L1425】

# 4. 重要度別の指摘

## 高

### 4.1 チェックアウト画面の注文確認情報が不足している

- **対象画面と要素**
  - `/checkout`
  - 注文内容リスト、合計、注文確定ボタン
- **発生する状態またはビューポート**
  - カートに商品が入っている状態
  - 全ビューポート
- **インターフェース上の問題**
  - チェックアウト画面では、商品名と数量、合計、注文確定ボタンのみが表示される。
  - 商品単価、商品カテゴリ、ライセンス、ダウンロード方法、購入後の入手先など、注文確定前に確認したい情報が画面内で不足している。
- **証拠と根拠**
  - チェックアウトのリストは `{商品名} × {数量}` のみで、単価やライセンスなどの補助情報は表示されていない。 【F:components/checkout/CheckoutContent.tsx†L70-L80】
  - 注文確定ボタンは表示されるが、確定前の確認項目は注意文 1 行に留まる。 【F:components/checkout/CheckoutContent.tsx†L80-L83】
- **情報理解または操作への影響**
  - ユーザーは「何を」「いくらで」「どの条件で」購入確定するのかを最終確認しにくい。
  - デジタル商品では、ファイル形式、ライセンス、ダウンロード期限、納品方法が購入判断に直結するため、注文確定前の不安が残る。
- **改善方向**
  - 注文内容を商品カードまたは注文行として再構成し、商品名、数量、単価、小計、ライセンス、ファイル形式、納品方法を表示する。
  - 合計欄の近くに「購入後はマイページのライブラリからダウンロード」などの次アクションを明示する。
  - 「注文確定」ボタンの直前に、確認済み項目をチェックリスト形式で置くと、確定操作の意味が明確になる。

### 4.2 商品カードの比較情報が少なく、一覧で選びにくい

- **対象画面と要素**
  - `/`
  - `/products`
  - カテゴリ、タグ、関連商品などで使用される `ProductCard`
- **発生する状態またはビューポート**
  - 商品一覧、トップページの商品セクション、関連商品
  - 全ビューポート
- **インターフェース上の問題**
  - 商品カードには画像、商品名、価格、詳細 CTA があるが、カテゴリ、ライセンス、ファイル形式、商用利用可否、容量などの比較軸が見えない。
  - お気に入りボタンは画像上にあるが、カード内の情報階層として「なぜこの商品を選ぶべきか」が弱い。
- **証拠と根拠**
  - 商品カードの表示要素は、画像、favorite、商品名、価格、詳細 CTA に限定されている。 【F:components/ProductCard.tsx†L31-L58】
- **情報理解または操作への影響**
  - 一覧で商品同士を比較する際、詳細ページに入らないと判断材料が不足する。
  - デジタル素材 EC では、用途、形式、ライセンスが価格と同じくらい重要なため、一覧での探索効率が落ちる。
- **改善方向**
  - カードにカテゴリバッジ、ファイル形式、ライセンス種別、主要タグ 1〜2 個を追加する。
  - 価格の近くに「商用利用可」「即時DL」などの判断ラベルを置く。
  - CTA は維持しつつ、カード全体の情報量を増やしすぎないよう、比較に必要な最小ラベルだけを追加する。

### 4.3 チャットの empty / reconnect / reauth 状態の次アクションが弱い

- **対象画面と要素**
  - `/chat/[conversationId]`
  - チャットヘッダー、メッセージリスト、エラー表示、送信フォーム
- **発生する状態またはビューポート**
  - メッセージ 0 件
  - `connecting`
  - `reconnecting`
  - `reauth-required`
  - 送信失敗
- **インターフェース上の問題**
  - 接続状態は文字で表示されるが、ユーザーが待つべきか、再ログインすべきか、再送すべきかの操作案内が弱い。
  - メッセージ 0 件の場合、メッセージリスト内に初回相談を促す empty state が表示されない。
  - `reauth-required` 時は送信ボタンが disabled になるが、次の明確な行動導線がフォーム周辺にない。
- **証拠と根拠**
  - チャットは状態をヘッダーの small 要素に表示する。 【F:components/chat/LineLikeChat.tsx†L252-L255】
  - エラー文は表示されるが、フォーム周辺の再ログイン導線や再試行 CTA はない。 【F:components/chat/LineLikeChat.tsx†L256-L280】
  - メッセージリストは `messages.map` のみで、0 件時の案内表示がない。 【F:components/chat/LineLikeChat.tsx†L261-L267】
- **情報理解または操作への影響**
  - ユーザーは、チャットが空なのか、読み込み中なのか、接続できていないのかを判断しにくい。
  - リアルタイム相談は購入不安の解消導線なので、状態が曖昧だと購入前の離脱につながりやすい。
- **改善方向**
  - メッセージ 0 件時に「商品について質問できます。例: ライセンス範囲を確認したい」などの empty state を表示する。
  - `reconnecting` 時は「再接続中です。送信内容は保持されています」などの待機説明を表示する。
  - `reauth-required` 時はログインページへの CTA を表示する。
  - 送信失敗時は「再送する」「入力内容を保持する」状態を明確にする。

### 4.4 商品詳細の購入アクションが情報量の後に出るため、主要 CTA の認識が遅れる可能性がある

- **発生する状態またはビューポート**

- **インターフェース上の問題**
  - 商品詳細では、説明、仕様、出品者情報の後に価格と CTA が配置される。
  - 購入判断に必要な仕様は重要だが、価格と購入操作が画面上部で即座に見えない可能性がある。
- **証拠と根拠**
  - 商品詳細は、商品名、説明、仕様、出品者情報、価格、CTA の順で構成されている。 【F:app/(store)/products/[slug]/page.tsx†L76-L115】

### 4.5 カートの商品行に小計がなく、数量変更と合計の関係が弱い

  - 全ビューポート
- **インターフェース上の問題**
  - 商品行には価格と数量はあるが、小計が表示されない。
  - 数量変更後に、各商品が合計にどう影響しているかを画面内で把握しにくい。
- **証拠と根拠**
  - カート行はカテゴリ、商品名、価格、数量、行操作で構成されている。 【F:components/cart/CartContent.tsx†L37-L45】

  - 複数商品購入時、どの商品が合計金額にどれだけ寄与しているかが分かりにくい。
  - 数量操作の結果を商品行単位で確認しにくい。
- **改善方向**
  - 商品行に「小計: ¥xxxx」を追加する。
  - 数量操作の近くに小計を置き、操作と結果の対応関係を強める。

# 8. インターフェース完成度: 要調整

## 総合判定

**要調整**

## 理由

- 商品探索、商品詳細、カート、チェックアウト、チャットという EC の主要画面は揃っている。
- 商品一覧の 2 カラム構造、商品カード、カート empty state、チェックアウト empty state、チャットの状態表示など、基礎的な画面内インターフェースは成立している。 【F:app/(store)/products/page.tsx†L69-L88】【F:components/cart/CartContent.tsx†L22-L55】【F:components/checkout/CheckoutContent.tsx†L57-L84】【F:components/chat/LineLikeChat.tsx†L251-L280】
- ただし、購入確定前の情報確認、商品一覧での比較情報、チャット状態の次アクション、モバイル時のフィルタ露出には調整余地がある。
- 特にチェックアウトの注文確認情報とチャットの状態設計は、購入操作・相談操作に直接影響するため、優先的に改善したい。

## 優先改善順

1. チェックアウトに商品単価、小計、ライセンス、納品方法、購入後導線を追加する。
2. 商品カードにカテゴリ、形式、ライセンス、主要タグなどの比較情報を追加する。
3. チャットに empty / reconnecting / reauth-required / send failed の次アクションを追加する。
4. モバイル商品一覧のフィルタを折りたたみ、商品結果への到達を早くする。
5. カート商品行に小計を追加し、数量操作と合計の関係を明確にする。
